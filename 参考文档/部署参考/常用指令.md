常用指令

一、AOSP 环境初始化

```bash
cd ~/aosp
source build/envsetup.sh
lunch aosp_x86_64-eng
```

二、编译相关

1. 编译 services.core

```bash
m services.core -j2
```

2. 编译 systemimage

```bash
m systemimage -j2
```

3. 整编

```bash
m -j2
```

三、模拟器与 adb

1. 查看设备

```bash
./out/host/linux-x86/bin/adb devices
```

2. 等待设备启动

```bash
./out/host/linux-x86/bin/adb wait-for-device
```

3. 重启设备

```bash
./out/host/linux-x86/bin/adb reboot
```

4. 关闭模拟器

```bash
./out/host/linux-x86/bin/adb emu kill
```

5. 清空 logcat

```bash
./out/host/linux-x86/bin/adb logcat -c
```

6. 清理模拟器运行态并重新启动（长命令）

```bash
cd ~/aosp
./out/host/linux-x86/bin/adb emu kill
rm -f out/target/product/generic_x86_64/userdata-qemu.img.qcow2
rm -f out/target/product/generic_x86_64/cache.img.qcow2
rm -f out/target/product/generic_x86_64/encryptionkey.img.qcow2
rm -rf out/target/product/generic_x86_64/tmpAdbCmds
source build/envsetup.sh
lunch aosp_x86_64-eng
emulator -verbose -cores 4 -show-kernel
```

四、服务验证

1. 查看服务是否注册

```bash
./out/host/linux-x86/bin/adb shell service list | grep permission
```

2. 查看服务状态

```bash
./out/host/linux-x86/bin/adb shell dumpsys permission_monitor
```

3. 查看权限监控服务日志

```bash
./out/host/linux-x86/bin/adb logcat -d | grep PermissionMonitorService
```

4. 查看 AppOpsScanner 调试日志

```bash
./out/host/linux-x86/bin/adb logcat -d | grep AppOpsScanner
```

5. 查看客户端与服务相关日志

```bash
./out/host/linux-x86/bin/adb logcat -d | grep -E "MainActivity|SettingsActivity|PermissionDetailActivity|PermissionMonitor|SecurityException|apppermissionmonitorservice"
```

五、触发权限使用测试

1. 启动系统相机

```bash
./out/host/linux-x86/bin/adb shell am start -a android.media.action.IMAGE_CAPTURE
```

2. 再次查看数据库统计

```bash
./out/host/linux-x86/bin/adb shell dumpsys permission_monitor
```

六、客户端 APK 构建

Windows PowerShell：

```powershell
.\gradlew.bat :app:assembleDebug
```

常见 APK 路径：

```text
d:\AppPermissionMonitorService\app\build\outputs\apk\debug\app-debug.apk
```

七、平台签名 APK 处理

1. 传输 APK 到 Ubuntu

```powershell
scp "d:\AppPermissionMonitorService\apk\app-debug.apk" topeet@192.168.147.128:/home/topeet/app-sign-test/app-debug.apk
```

2. 使用 AOSP 自带 Java 和 platform key 签名

```bash
LD_LIBRARY_PATH=/home/topeet/aosp/out/host/linux-x86/lib64 \
/home/topeet/aosp/prebuilts/jdk/jdk8/linux-x86/bin/java \
-Djava.library.path=/home/topeet/aosp/out/host/linux-x86/lib64 \
-jar /home/topeet/aosp/out/host/linux-x86/framework/signapk.jar \
/home/topeet/aosp/build/target/product/security/platform.x509.pem \
/home/topeet/aosp/build/target/product/security/platform.pk8 \
/home/topeet/app-sign-test/app-debug.apk \
/home/topeet/app-sign-test/app-platform-signed.apk
```

3. 安装签名后的 APK

```bash
cd ~/aosp
./out/host/linux-x86/bin/adb uninstall com.example.apppermissionmonitorservice
./out/host/linux-x86/bin/adb install /home/topeet/app-sign-test/app-platform-signed.apk
```

八、常用排查命令

1. 查看 BOOT FAILURE 上下文

```bash
./out/host/linux-x86/bin/adb logcat -d | grep -A 40 -B 10 "BOOT FAILURE starting PermissionMonitorService"
```

2. 查看 SELinux / ServiceManager 相关错误

```bash
./out/host/linux-x86/bin/adb logcat -d | grep -E "permission_monitor|ServiceManager|avc: denied"
```

3. 查看系统属性与开机状态

```bash
./out/host/linux-x86/bin/adb shell getprop | grep boot
```

十、Linux 系统用户信息查询（VMware 虚拟机内）

1. 查看当前用户信息

```bash
id
```

2. 查看当前用户名

```bash
whoami
```

3. 查看当前登录用户列表

```bash
w
```

4. 查看系统所有用户

```bash
cat /etc/passwd
```

5. 查看当前用户所属组

```bash
groups
```

6. 查看指定用户信息

```bash
id <用户名>
```

示例：
```bash
id topeet
```

7. 查看当前用户的环境变量

```bash
echo $USER
echo $HOME
echo $UID
```

8. 查看网络配置和 IP 地址

```bash
# 查看所有网络接口信息
ip addr show

# 查看活跃网络接口（更简洁）
ip addr show | grep inet

# 查看 eth0 接口的 IP
ip addr show eth0

# 使用 ifconfig（传统方式，需安装 net-tools）
ifconfig
```

9. 查看网关和路由信息

```bash
ip route show
```

10. 测试网络连通性

```bash
# 测试外网连通性
ping -c 3 www.google.com

# 测试局域网连通性
ping -c 3 192.168.1.1
```

十一、VMware 虚拟机空间清理

用于清理 VMware 虚拟机磁盘空间，回收未使用的存储空间。

1. 查看虚拟机磁盘列表

```bash
sudo vmware-toolbox-cmd disk list
```

2. 收缩磁盘空间（清理未使用空间）

```bash
sudo vmware-toolbox-cmd disk shrink /
```

> **说明**：执行收缩前建议先在虚拟机内进行磁盘碎片整理（Windows）或使用 `fstrim`（Linux），以获得更好的空间回收效果。
