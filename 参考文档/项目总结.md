# 应用权限监控服务项目总结

## 一、项目概述

本项目是一个**安卓框架开发综合实训**课题，目标是开发一个系统级的**应用权限监控服务（App Permission Monitor Service）**，用于实时监控应用中敏感权限的使用情况。

### 1.1 项目目标

| 目标维度 | 具体内容 |
|---------|---------|
| **核心功能** | 实时监控、历史记录、查询接口、回调注册、权限控制 |
| **技术要求** | SystemService、Binder、AIDL、Parcelable、SQLite |
| **性能要求** | 不影响系统流畅度、稳定性、数据安全性 |

### 1.2 技术栈

- **Framework层**: SystemService、Binder、AIDL、Parcelable、HandlerThread、SQLite
- **应用层**: Service绑定、权限声明、UI开发（RecyclerView）
- **系统工具**: Logcat、dumpsys、adb

## 二、系统架构

### 2.1 整体架构

```
[客户端App] <--Binder--> [PermissionMonitorService] <---> [SQLite数据库]
                              |
                              v
                    [AppOpsManager/PackageManager]
```

### 2.2 模块划分

| 模块 | 职责 | 关键文件 |
|------|------|---------|
| **服务模块** | Binder接口实现、数据采集、回调管理 | PermissionMonitorService.java |
| **监控管理器** | 权限监控逻辑调度 | PermissionMonitorManager.java |
| **数据采集** | AppOps扫描、权限记录检测 | AppOpsScanner.java |
| **数据存储** | SQLite数据库操作 | PermissionDatabase.java, PermissionRepository.java |
| **AIDL接口** | 跨进程通信定义 | IPermissionMonitor.aidl, IPermissionCallback.aidl |
| **数据实体** | Parcelable数据传输对象 | PermissionRecord.java |

## 三、核心实现

### 3.1 服务端核心组件

#### 3.1.1 PermissionMonitorService
- 继承SystemService，运行在SystemServer进程中
- 注册到ServiceManager，服务名为`permission_monitor`
- 管理RemoteCallbackList实现多客户端回调

#### 3.1.2 AppOpsScanner
- 使用AppOpsManager获取权限操作记录
- HandlerThread定期扫描（每秒）
- 对比上次扫描结果，发现新记录触发回调

#### 3.1.3 数据持久化
- SQLite数据库存储权限使用记录
- 表结构：id, package, permission, timestamp, extra
- 支持定期清理旧数据（保留最近7天）

### 3.2 AIDL接口定义

**IPermissionMonitor.aidl** 主要方法：
- `getRecords(String packageName, long startTime, long endTime)` - 查询指定应用的权限记录
- `getAllRecords(long startTime, long endTime)` - 查询所有权限记录
- `registerCallback(IPermissionCallback callback)` - 注册实时回调
- `unregisterCallback(IPermissionCallback callback)` - 注销回调
- `setMonitoredPermissions(List<String> permissions)` - 设置监控权限类型
- `getMonitoredPermissions()` - 获取当前监控权限列表

### 3.3 权限控制

- 自定义权限：`com.android.permission.ACCESS_PERMISSION_MONITOR`
- 保护级别：`signature`（系统签名应用可访问）
- 在Binder接口方法中检查调用者权限

## 七、集成与验证流程

### 7.1 AOSP集成步骤

```bash
# 1. 复制服务代码到AOSP
cp -r aosp/java/com/android/server/permissionmonitor/* \
    ~/aosp/frameworks/base/services/core/java/com/android/server/permissionmonitor/

# 2. 修改Android.bp添加编译配置
# 3. 修改SystemServer.java添加服务启动
# 4. 修改AndroidManifest.xml添加权限定义
# 5. 修改strings.xml添加字符串资源

# 6. 编译
source build/envsetup.sh
lunch aosp_x86_64-eng
make services.core -j2

# 7. 启动模拟器验证
emulator
```

### 7.2 验证命令
