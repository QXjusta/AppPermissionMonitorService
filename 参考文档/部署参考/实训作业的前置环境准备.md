# 实训作业的前置环境准备

## 一、文档目的

本文档用于整理安卓框架开发综合实训在正式开发前必须完成的环境准备工作，适用于以下实训方向：

- 自定义系统服务开发
- AOSP Framework 层调试
- Android 10 系统源码编译
- 模拟器与系统镜像验证
- 客户端与系统服务联调

结合当前实训课题 [参考文档.md](file:///d:/AppPermissionMonitorService/参考文档/参考文档.md) 的课程目标，本阶段重点不是马上写业务代码，而是先把 **宿主机、VMware 虚拟机、Ubuntu 编译环境、Android 10 源码、模拟器硬件加速、VMware Tools** 这些基础条件全部准备好。

只有前置环境稳定，后续的 AOSP 编译、SystemServer 改造、SELinux 策略修改、客户端调试才不会频繁卡在环境问题上。

---

## 二、为什么前置环境很重要

在实训过程中，最常见的阻塞问题往往不是代码本身，而是环境没有准备好，例如：

- VMware 没开 CPU 虚拟化，模拟器根本跑不起来
- Ubuntu 没装好 VMware Tools，文件传输和分辨率很差
- Android 10 源码没有正确同步，后面无法编译
- 宿主机 BIOS 没开 VT-x / AMD-V，虚拟化加速失效
- Windows 的 Hyper-V 和 VMware / 模拟器冲突
- 编译环境依赖缺失，`make` 或 `repo sync` 中途失败

所以这份文档的目标是先把这些问题统一提前解决。

## 四、宿主机前置检查

正式搭建虚拟机之前，先检查宿主机环境。

### 4.1 检查 CPU 是否支持虚拟化

在 Windows PowerShell 中执行：

```powershell
systeminfo | findstr /C:"Virtualization Enabled In Firmware"
```

如果能看到类似：

```text
Virtualization Enabled In Firmware: Yes
```

说明 BIOS 已启用虚拟化。

### 4.2 BIOS / UEFI 中开启虚拟化

如果没有启用，需要重启电脑进入 BIOS/UEFI，找到以下选项并开启：

| CPU 厂商 | 选项名称 |
|----------|----------|
| Intel | `Intel Virtualization Technology (VT-x)` |
| AMD | `AMD-V` / `SVM Mode` |

常见进入 BIOS 的按键：
- `Del`
- `F2`
- `F10`
- `F12`

### 4.3 关闭 Hyper-V 冲突项

如果宿主机是 Windows，建议关闭会和 VMware / Android 模拟器冲突的组件：

```powershell
Disable-WindowsOptionalFeature -Online -FeatureName Microsoft-Hyper-V-All
Disable-WindowsOptionalFeature -Online -FeatureName VirtualMachinePlatform
Disable-WindowsOptionalFeature -Online -FeatureName HypervisorPlatform
```

如仍有冲突，再执行：

```powershell
bcdedit /set hypervisorlaunchtype off
shutdown /r /t 0
```

## 五、VMware 虚拟机准备

这部分主要参考你给出的配置方向，适用于实训中常见的 VMware Workstation 环境。

### 5.1 新建 Ubuntu 虚拟机

推荐在 VMware 中新建一台 Ubuntu 18.04 / 20.04 虚拟机，用来承担 AOSP 编译任务。

推荐配置：

| 配置项 | 建议值 |
|--------|--------|
| CPU | 4 核以上 |
| 内存 | 至少 8GB，推荐 12GB-16GB |
| 磁盘 | 至少 150GB，推荐 200GB+ |
| 网络 | NAT 或桥接 |
| 显示 | 默认即可 |

### 5.2 必须打开 CPU 虚拟化透传

关闭虚拟机后，进入：

`编辑虚拟机设置 -> 处理器`

勾选以下选项：

- `虚拟化 Intel VT-x/EPT 或 AMD-V/RVI`
- `虚拟化 CPU 性能计数器`

这是后续在虚拟机里再跑 Android 模拟器时非常关键的一步。

### 5.3 建议的内存与磁盘规划

AOSP 编译很吃资源，建议：

- **最低可用**：4 核 CPU + 8GB 内存 + 150GB 磁盘
- **较稳定**：6-8 核 CPU + 12GB~16GB 内存 + 200GB 磁盘

如果磁盘太小，后面 `repo sync` 和 `make` 很容易失败。

### 5.4 网络模式建议

| 模式 | 场景 | 建议 |
|------|------|------|
| NAT | 普通联网、下载源码 | 推荐 |
| 桥接 | 需要和局域网设备互通、scp 传文件 | 也可用 |

如果你后续要把 APK 从 Windows 传到 Ubuntu，桥接会更直观，但 NAT 也可以配合 ssh/scp 使用。

## 六、Ubuntu 系统基础配置

### 6.1 安装系统后先更新

```bash
sudo apt update
sudo apt upgrade -y
```

### 6.2 安装常用开发工具

```bash
sudo apt install -y git curl vim wget unzip zip rsync openssh-server net-tools
```

### 6.3 安装 AOSP 常见依赖

以 Ubuntu 为例，建议先安装常见编译依赖：

```bash
sudo apt install -y \
    bc bison build-essential ccache curl flex g++-multilib gcc-multilib \
    git gnupg gperf imagemagick lib32ncurses5-dev lib32readline-dev \
    lib32z1-dev liblz4-tool libncurses5 libncurses5-dev libsdl1.2-dev \
    libssl-dev libxml2 libxml2-utils lzop pngcrush rsync schedtool \
    squashfs-tools xsltproc zip zlib1g-dev openjdk-8-jdk python
```

不同 Ubuntu 版本的包名可能有少量差异，如果个别包安装失败，需要按当前系统版本调整。

### 6.4 配置 SSH 服务

后续你可能需要从 Windows 往虚拟机传 APK 或源码补丁，建议开启 ssh：

```bash
sudo systemctl enable ssh
sudo systemctl start ssh
```

查看虚拟机 IP：

```bash
ip addr show | grep inet
```

查看当前用户名：

```bash
whoami
id
```

## 七、VMware Tools 安装与检查

为了避免虚拟机分辨率、拖放、剪贴板和文件传输问题，建议先把 VMware Tools 处理好。

### 7.1 先检查是否已经安装

在 Ubuntu 中执行：

```bash
systemctl status vmtoolsd
vmware-toolbox-cmd -v
dpkg -l | grep vmware
lsmod | grep vmw
```

如果服务正常运行，并且能查到版本信息，通常说明已经安装。

### 7.2 推荐安装方式

优先使用开源版 `open-vm-tools`：

```bash
sudo apt update
sudo apt install -y open-vm-tools open-vm-tools-desktop
sudo systemctl restart vmtoolsd
```

### 7.3 不建议重复安装

不要在已经安装了 `open-vm-tools` 的基础上，再去重复安装 VMware 官方 `vmware-tools`，否则可能导致：

- 服务冲突
- 图形异常
- 拖放异常
- 内核模块冲突
- 系统不稳定甚至崩溃

### 7.4 安装完成后验证

```bash
systemctl status vmtoolsd
vmware-toolbox-cmd -v
```

同时检查这些功能是否正常：

- 分辨率自适应
- 鼠标移动顺滑
- 复制粘贴
- 宿主机与虚拟机之间拖放文件

## 八、CPU 虚拟化与模拟器运行环境

如果后续要在 Ubuntu 虚拟机里启动 Android 模拟器，就必须保证虚拟化链路是通的：

**宿主机 BIOS 虚拟化开启**
→ **VMware 处理器设置中打开 VT-x/EPT / AMD-V/RVI**
→ **Ubuntu 中具备 KVM 或等效加速支持**

### 8.1 在 Linux 中检查虚拟化支持

```bash
grep -E "(vmx|svm)" /proc/cpuinfo
```

### 8.2 安装 KVM 相关组件

```bash
sudo apt-get update
sudo apt-get install -y qemu-kvm libvirt-daemon-system libvirt-clients bridge-utils
sudo adduser $USER kvm
```

### 8.3 检查 KVM 是否可用

```bash
egrep -c '(vmx|svm)' /proc/cpuinfo
kvm-ok
```

如果 `kvm-ok` 不可用，可以安装：

```bash
sudo apt install cpu-checker
```

### 8.4 常见问题

如果模拟器启动失败，优先检查：

1. 宿主机 BIOS 是否开启 VT-x / AMD-V
2. VMware 是否勾选虚拟化透传
3. Windows 是否残留 Hyper-V 冲突
4. Ubuntu 是否装好 KVM 相关依赖

## 九、Android 10 源码下载准备

这一部分是后续做系统服务开发的核心前置条件。

### 9.1 为什么要下载 Android 10 源码

因为实训后面会涉及：

- 修改 `SystemServer.java`
- 新增系统服务类
- 修改 `AndroidManifest.xml`
- 修改 `strings.xml`
- 修改 `service.te` 和 `service_contexts`
- 编译 systemimage 或完整镜像

这些都必须建立在本地有完整 AOSP 源码的前提下。

### 9.2 创建工作目录

建议在 Ubuntu 中使用独立目录：

```bash
mkdir -p ~/aosp
cd ~/aosp
```

### 9.3 安装 repo 工具

```bash
mkdir -p ~/bin
curl https://storage.googleapis.com/git-repo-downloads/repo -o ~/bin/repo
chmod a+x ~/bin/repo
export PATH=~/bin:$PATH
```

建议把 PATH 写入 `~/.bashrc`：

```bash
echo 'export PATH=~/bin:$PATH' >> ~/.bashrc
source ~/.bashrc
```

### 9.4 初始化 Android 10 源码仓库

Android 10 常见分支可以使用：

```bash
repo init -u https://android.googlesource.com/platform/manifest -b android-10.0.0_r47
```

如果你们课程或老师指定了特定 tag，就以指定 tag 为准。

### 9.5 同步源码

```bash
repo sync -j4
```

说明：
- `-j4` 表示 4 线程同步
- 网络一般时不要开太大，否则容易失败

### 9.6 源码下载注意事项

1. AOSP 源码体积非常大，下载时间较长
2. 下载过程中可能因为网络波动失败，需要重复执行 `repo sync`
3. 磁盘空间必须充足
4. 建议在网络稳定时操作

### 9.7 下载完成后的基本检查

```bash
ls
```

正常应能看到类似目录：

```text
art
bionic
build
frameworks
packages
system
vendor
```

## 十、Android 10 编译前的环境确认

### 10.1 配置编译环境

```bash
cd ~/aosp
source build/envsetup.sh
lunch aosp_x86_64-eng
```

### 10.2 建议先做一次基础编译验证

```bash
make -j2
```

如果机器性能更好，也可以逐步提高并发。

### 10.3 编译前要确认的三件事

后面做 AOSP 问题排查时，一定要同时对齐这三件事：

1. **源码改的是哪一套**
2. **当前 shell 编的是哪套 target**
3. **模拟器跑的是哪套镜像**

只要这三件事不一致，就很容易出现：

- 明明代码改了但系统没变化
- 编译成功但功能不生效
- 不同终端下编译结果不一致

- adb 可用

- 调试 Binder 调用

## 十四、建议的实际操作顺序

建议按下面顺序做，不容易乱：

1. 宿主机 BIOS 开虚拟化
2. Windows 关闭 Hyper-V 冲突
3. VMware 新建 Ubuntu 虚拟机
4. VMware 中打开 CPU 虚拟化透传
5. Ubuntu 安装基础依赖
6. 安装并验证 VMware Tools
7. 检查 Linux 虚拟化和 KVM
8. 安装 repo 工具
9. 下载 Android 10 源码
10. 配置 `envsetup.sh` + `lunch`
11. 做一次基础编译验证
12. 再开始后续系统服务开发

## 十五、总结

实训作业能不能顺利推进，往往取决于前置环境是否稳定。

你后面要做的系统服务开发、AOSP 编译、SELinux 修改、客户端联调，本质上都依赖下面这条环境链：

```text
宿主机 BIOS 虚拟化
→ Windows / VMware 配置正确
→ Ubuntu 编译环境稳定
→ VMware Tools 正常
→ Android 10 源码下载完成
→ AOSP 编译链可用
→ 模拟器可正常启动
```

只要这条链路通了，后面的实训就会顺很多。

如果后续你要继续细化，还可以把这份文档再拆成：
- 虚拟机搭建篇
- Android 10 源码下载篇
- 编译环境验证篇
- 模拟器运行篇

不过对于当前实训来说，先把它整理成一份总的前置环境文档是最合适的。
