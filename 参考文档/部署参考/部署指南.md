# 应用权限监控服务 - 完整部署指南

## 一、概述

本文档详细描述如何在全新环境中从头部署 **App Permission Monitor Service** 项目，包括 AOSP 环境搭建、服务集成、系统编译和客户端部署的完整流程。

---

## 二、环境准备

### 2.1 硬件要求

| 组件 | 最低要求 | 推荐配置 |
|------|---------|---------|
| CPU | 4核心 | 8核心以上 |
| 内存 | 16GB | 32GB以上 |
| 存储 | 200GB可用空间 | 300GB以上（SSD） |

### 2.2 操作系统要求

- **Ubuntu 20.04 LTS** 或 **Ubuntu 22.04 LTS**（推荐）
- 确保系统已更新：

```bash
sudo apt update && sudo apt upgrade -y
```

### 2.3 安装依赖工具

```bash
# 安装基础依赖
sudo apt-get install -y openjdk-17-jdk git gnupg flex bison gperf build-essential \
    zip curl zlib1g-dev gcc-multilib g++-multilib libc6-dev-i386 \
    libncurses5-dev libncurses5 libx11-dev libreadline-dev libgl1-mesa-dev \
    libxml2-utils xsltproc unzip fontconfig python3 python3-pip

# 安装 Repo 工具
mkdir ~/bin
curl https://mirrors.bfsu.edu.cn/git/git-repo -o ~/bin/repo
chmod +x ~/bin/repo
export PATH=~/bin:$PATH
```

## 七、启动模拟器验证

### 7.1 启动模拟器

重新编译过系统镜像后，使用这个指令清理模拟器数据，并且重新启动模拟器：
```bash
cd ~/aosp
./out/host/linux-x86/bin/adb emu kill
rm -f out/target/product/generic_x86_64/userdata-qemu.img.qcow2
rm -f out/target/product/generic_x86_64/cache.img.qcow2
rm -f out/target/product/generic_x86_64/encryptionkey.img.qcow2
rm -rf out/target/product/generic_x86_64/tmpAdbCmds
source build/envsetup.sh
lunch aosp_x86_64-eng
emulator -verbose -cores 4 -show-kernel
```
若并没有再次修改系统镜像，，直接启动模拟器即可。
启动命令为：
```bash
cd ~/aosp
source build/envsetup.sh
lunch aosp_x86_64-eng
emulator -verbose -cores 4 -show-kernel
```

### 7.2 验证服务启动

打开新终端，执行以下命令：

```bash
# 检查服务是否注册
./out/host/linux-x86/bin/adb shell service list | grep permission

# 查看服务状态
./out/host/linux-x86/bin/adb shell dumpsys permission_monitor

# 查看服务日志
./out/host/linux-x86/bin/adb logcat -s PermissionMonitorService
```

## 八、客户端应用部署

### 8.1 编译客户端 APK

在 Windows 环境下：
注意：app文件位置根据自己的电脑路径进行调整，下面都用的是示例路径
```powershell
cd AppPermissionMonitorService
.\gradlew.bat :app:assembleDebug
```

APK 输出路径：
```
AppPermissionMonitorService\app\build\outputs\apk\debug\app-debug.apk
```

在本工程文件下提前编译好的测试apk路径为：
```
AppPermissionMonitorService\apk\app-debug.apk
```

### 8.2 平台签名

由于权限保护级别是 `signature`，需要使用系统签名：

```bash
# 传输 APK 到 Ubuntu 用户名ip根据自己情况填写，如何获取用户名和ip地址请参考常用指令.md
scp "d:\AppPermissionMonitorService\app\build\outputs\apk\debug\app-debug.apk" \
    user@ubuntu-ip:/home/user/app-debug.apk

# 使用 AOSP 签名 这里使用的是项目中的apk文件，具体的路径和名称还得根据实际情况调整
cd ~/aosp
LD_LIBRARY_PATH=out/host/linux-x86/lib64 \
prebuilts/jdk/jdk8/linux-x86/bin/java \
-Djava.library.path=out/host/linux-x86/lib64 \
-jar out/host/linux-x86/framework/signapk.jar \
build/target/product/security/platform.x509.pem \
build/target/product/security/platform.pk8 \
/home/user/app-debug.apk \
/home/user/app-platform-signed.apk
```

### 8.3 安装客户端应用
# 先卸载旧版本再安装新版本，如果没旧版本可以直接安装      这里同样使用的是项目中的apk文件，具体的路径和名称还得根据实际情况调整
```bash
cd ~/aosp
./out/host/linux-x86/bin/adb uninstall com.example.apppermissionmonitorservice
./out/host/linux-x86/bin/adb install /home/topeet/app-sign-test/app-platform-signed.apk
```

## 十、常见问题排查

### 10.1 编译错误 - 找不到类

**原因**：AIDL 文件未正确编译

**解决**：
```bash
make clean-services
make services.core -j4
```

### 10.2 服务未启动

**原因**：SystemServer 中未正确添加服务启动代码

**解决**：检查 `SystemServer.java` 中的修改是否正确

### 10.3 权限拒绝

**原因**：客户端应用未签名或签名不匹配

**解决**：确保使用平台签名，或修改权限保护级别为 `normal`

### 10.4 内存不足导致编译失败

**解决**：
```bash
# 降低并发数
make services.core -j2

# 增加交换空间
sudo fallocate -l 16G /swapfile
sudo chmod 600 /swapfile
sudo mkswap /swapfile
sudo swapon /swapfile
```

### 10.5 模拟器启动失败

**解决**：
```bash
# 清理旧镜像
./out/host/linux-x86/bin/adb emu kill
rm -f out/target/product/generic_x86_64/userdata-qemu.img.qcow2
rm -f out/target/product/generic_x86_64/cache.img.qcow2

# 重新启动
emulator -verbose -cores 2 -memory 4096
```
